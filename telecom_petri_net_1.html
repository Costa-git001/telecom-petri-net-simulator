Use Case Description

Modern telecommunications networks must simultaneously handle diverse traffic streams — voice calls, data sessions, video streams, and emergency priority calls — across finite shared bandwidth. This system models a Next-Generation Telecom Core Network that receives incoming call/session requests and routes them through a multi-stage processing pipeline: admission control, traffic classification, bandwidth allocation, quality-of-service enforcement, and release/termination.

The network enforces strict priority queuing: Emergency (E) > Voice (V) > Video (T) > Data (D). Each traffic class consumes a different bandwidth slice from a shared pool. If bandwidth is exhausted, lower-priority sessions are pre-empted or queued. Congestion triggers a renegotiation sub-process where sessions may be downgraded or dropped with a notification sent to the user.

The Petri net models the full lifecycle: Arrival → Classification → Queue → Admission Decision → Active Session → Monitoring → Release → Bandwidth Return. Concurrent sessions, conflict resolution, and bandwidth recycling are all captured. A hierarchical decomposition separates the top-level orchestration from the detailed sub-nets for each traffic class handler.

Traffic Classes

Emergency (E) — Pre-empts all, guaranteed QoS, 20 BU/session
Voice (V) — Real-time, low latency, 5 BU/session
Video (T) — Streaming, adaptive bitrate, 10 BU/session
Data (D) — Best-effort, elastic, 3 BU/session

Petri Net Scope

• Top-level HPN: 10 transitions, 12 places — orchestrates all traffic flows

• Sub-nets: Admission Control, Bandwidth Manager, QoS Monitor (each with 4+ transitions)

• CPN: Tokens carry color tuples (class, id, BU_demand, priority)

• Total arcs: 28+ in top net; guard functions enforce priority rules

• Properties verified: Reachability, Boundedness, Liveness, Safeness, Reversibility, Fairness, Coverability

4 Liveness A transition t is live if for every reachable marking M, there exists a reachable M' from M where t can fire. Net is live if all transitions are live. ✓ L3-live (quasi-live) Every transition fires at least once in each simulation run (quasi-live). T_Drop fires only under congestion — it is not L4-live (dead-lock free) because the net can idle. Auto-run shows all 10 transitions fire across random injections. All processing paths (admit, drop, release, preempt, monitor) remain reachable throughout operation. No part of the network permanently stalls.
5 Reversibility A net is reversible if from every reachable marking M, the initial marking M₀ is reachable. ✓ YES After session release (T_Release), BW tokens return to P_BW_Pool and session places empty — returning to M₀. The Reset button demonstrates this programmatically. After all sessions end, the network returns to idle state — no permanent resource exhaustion. Essential for 24/7 telecom uptime.
6 Fairness A transition t is fair if it cannot be indefinitely ignored while other transitions fire. Two transitions are in conflict if they share an input place. ~ BOUNDED FAIR Voice and Data compete for BW_Pool. Priority guard prevents starvation of Emergency calls. However, Data sessions may be starved when BW is saturated by Voice/Video. Stats panel shows per-class session counts — imbalance signals unfairness. Emergency and Voice are always fairly served. Data sessions face bounded unfairness (intentional QoS policy) — this is a design choice, not a flaw.
7 Coverability A marking M is coverable if there exists a reachable M' ≥ M (componentwise). Used to check if a resource level can be reached. ✓ YES Coverability tree shows that any bandwidth allocation ≤50 BU is coverable. The karpagel tree (Karp-Miller) can be constructed — all places are ω-bounded relative to BW_Pool capacity. Any valid bandwidth consumption pattern (e.g., 3 video + 1 voice = 35 BU) is coverable from the initial state, confirming the model is complete.
8 Deadlock-Freedom A marking is a deadlock if no transition is enabled. A net is deadlock-free if no reachable marking is a deadlock. ~ CONDITIONAL Deadlock occurs only when BW_Pool = 0 AND all queues are full AND no session is completing. This is a controlled deadlock (congestion state), resolved by T_Timeout or T_Preempt firing. Simulation logs "DEADLOCK RISK" when all transitions are disabled. Network congestion is the only deadlock scenario. The pre-emption and timeout transitions act as deadlock-resolution mechanisms — matching real PSTN behaviour.
9 Persistence A net is persistent if for any two enabled transitions, firing one does not disable the other. ✗ NO T_Admit_Voice and T_Admit_Video both compete for BW_Pool tokens. Firing one may disable the other when BW is scarce — demonstrating conflict/choice in the net. Resource contention is inherently non-persistent — two call admissions compete for the same bandwidth. This models realistic resource conflict.
10 Structural Boundedness A net is structurally bounded if it is bounded for every finite initial marking. ✓ YES The incidence matrix C shows no positive circuits that could cause unbounded growth. P-invariants confirm token conservation across BW cycle (P_BW_Pool + Σ P_Active_* = 50 BU). The bandwidth conservation law is a P-invariant: total BU in pool + total BU in active sessions = 50 always. This is the fundamental network resource constraint.

Limitations of the Petri Net Model

While Petri nets are powerful for modeling concurrency and resource management, several aspects of a real telecom system cannot be fully or accurately captured:

Structural Limitations

1. State Space Explosion: The reachability graph grows exponentially with concurrent sessions and traffic classes. With 50 BW units and 4 traffic classes, the state space becomes intractable to enumerate manually. Only simulation (not exhaustive analysis) is feasible here.

2. Time Abstraction: Standard Petri nets are untimed. Real telecom systems have critical timing constraints: call setup latency, hold timers, retransmission delays. Timed or Stochastic Petri Nets (SPNs) would be needed, adding complexity.

3. No Probabilistic Behavior: Token injection in this model is deterministic or user-controlled. Real call arrival follows Poisson distributions; service times are exponential. SPNs with negative-exponential firing rates would be needed for accurate performance analysis.

4. Hierarchy Depth: The substitution transitions (admission, bandwidth manager) are only expanded one level deep. A real system would require 5–8 levels of decomposition for full fidelity.

Semantic Limitations

5. No Continuous Variables: Bandwidth is modeled discretely (integer units). Real networks use continuous-valued bandwidth, jitter, and signal-to-noise ratios — requiring hybrid Petri nets or differential equations.

6. No Fault Modeling: Link failures, node crashes, and packet loss are not modeled. A complete resilience model would require additional places/transitions for failure and recovery paths.

7. Geographic Topology: The net treats the network as a single node. Real telecom networks are geographically distributed (BSC, MSC, PSTN gateway chains) — requiring colored tokens with location attributes and routing logic.

8. Security & Authentication: SIM authentication, encryption negotiation, and rogue-call rejection are absent. These sub-processes would add significant complexity to the admission sub-net.

9. Fairness Enforcement Gap: The model captures fairness as a property but cannot enforce weighted-fair-queuing (WFQ) algorithms directly — these require numerical weight computation beyond arc annotations.
